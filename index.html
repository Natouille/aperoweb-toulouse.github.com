Des CSS pour le futur

L'écosystème de CSS continue d'évoluer et d'innover pour adresser les problèmes actuels que rencontrent les développeurs d'application web : syntaxe en évolution constante, modularité, automatisation. L'usage de pré-processeurs comme Sass a pointé du doigt les manques de CSS mais ne permet pas pour autant de pouvoir utiliser des fonctionnalités non encore standardisées comme CSS Grid Layout. Heureusement des outils existent pour faciliter le travail des intégrateurs et des développeurs front-end.

PostCSS à l'instar de Babel en JavaScript permet de transformer et d’étendre la syntaxe CSS. Les équipes de Google, Twitter ou Shopify ont toutes adopté ces outils pour minimiser leur dette technique et pour utiliser dès aujourd’hui la syntaxe de demain. Vous aussi bénéficiez du travail de la communauté et demandez en plus à vos CSS.

Présenté par Maxime Thirouin, développeur freelance, contribue sous le nom de MoOx dans la communauté open-source. Il adore bricoler le CSS et le JS autant qu'il aime jardiner et s'occuper de son potager bio.
